Práticas 2.2 — CRUD com PreparedStatement

Por que PreparedStatement?

❌ Statement (Vulnerável)

String sql = "SELECT * FROM aluno WHERE nome = '" 
           + nome + "'";
// Se nome = "'; DROP TABLE aluno; --"
// SQL Injection!

✅ PreparedStatement (Seguro)

String sql = "SELECT * FROM aluno WHERE nome = ?";
PreparedStatement ps = conn.prepareStatement(sql);
ps.setString(1, nome);
// Parâmetro escapado automaticamente!
⚠️ SQL Injection — Ataque onde o invasor insere comandos maliciosos através de entradas do usuário. SEMPRE use PreparedStatement com parâmetros.

Vantagens do PreparedStatement:

  • ✅ Segurança — Previne SQL Injection
  • ✅ Performance — Query compilada uma vez, executada várias
  • ✅ Legibilidade — SQL mais limpo sem concatenação

Operações CRUD

INSERT — Inserir registro

String sql = "INSERT INTO aluno (nome, email) VALUES (?, ?)";
PreparedStatement ps = conn.prepareStatement(sql);
ps.setString(1, "João");      // primeiro ?
ps.setString(2, "joao@email.com"); // segundo ?
int rows = ps.executeUpdate();  // retorna linhas afetadas
